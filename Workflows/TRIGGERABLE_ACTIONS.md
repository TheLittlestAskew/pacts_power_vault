# Workflows — Triggerable Actions Index (Pacts & Power)

Every action you can trigger from the `Workflows/` folder: what it does, and how to fire it.
Generated by auditing each file.

> **Note:** Pacts & Power is a **complete** campaign (ended 2026-05-31) and this vault has **no
> pipeline watcher** (unlike SITL/Ashfall). The workflow here is a set of **manually-run**
> scripts plus the roll-sync extension and the Convo instruction docs — there is no auto-on-new-
> recording automation, status window, or approve flow.

**Path shorthand:** `~` = `C:\Users\theli\pacts_power_vault`. Scripts live in `~\Workflows`.

## 1. Transcription

### `pp_transcribe.js`
- **What:** Transcribes a Pacts & Power session recording with AssemblyAI (campaign vocabulary
  boost + custom spelling from the spelling dictionary, speaker diarization).
- **Trigger:**
  - `node pp_transcribe.js` — interactive picker over `~\Session_Sources\Recordings\`
  - `node pp_transcribe.js session13.mp3` — by filename
- **Run from:** `~\Workflows`  ·  **Needs:** `ASSEMBLYAI_API_KEY` in a `.env` at the Workflows
  folder, and `npm install dotenv`. (Per the script header.)

---

## 2. Transcript post-processing (Python)

### `pacts_spelling_corrector.py`
- **What:** Applies canonical spelling corrections from the campaign dictionary
  (`PACTS_POWER_SPELLING_DICTIONARY.md`) to raw transcripts.
- **Trigger:** `python pacts_spelling_corrector.py [--input-dir DIR] [--output-dir DIR] [--dry-run] [--file FILENAME]`
  - `--dry-run` previews without writing; `--file` does a single file.
- **Run from:** `~\Workflows`

### `pacts_notes_normalizer.py`
- **What:** Normalizes condensed `**Bold Label.**` session notes (Session 04+) into the full v8
  template structure (`## Section N — Title` + `### Subheading`) so they parse uniformly for the
  Rectrix Caedere session page. Structure only — never invents or deletes content.
- **Trigger:** `python pacts_notes_normalizer.py [--dry-run] [--file <path relative to vault root>]`
- **Run from:** `~\Workflows`
- **See also:** `pacts_spelling_corrector.py run code.md` — saved run-command notes.

## 3. DDB party character sheets → vault

### `ddb_party_sync.js`
- **What:** Pulls the party's full DDB character sheets and writes raw JSON + a readable markdown
  sheet into `~\03-Characters\01 PCs\Party Character Sheets\`, named by real character name. A
  **one-time archive** (campaign complete; no watcher). Anonymous by default; set `DDB_COBALT` in
  `~\.env` to also fetch Campaign-Only sheets (Sanis).
- **Trigger:** `node ddb_party_sync.js`  ·  **Run from:** `~\Workflows`
- **Details + roster/IDs:** see `DDB_PARTY_SYNC.md`. Output is **gitignored** (public repo —
  other players' data kept local-only).

---

## 4. DDB dice rolls → Supabase

### `ddb-roll-sync\` — Chrome extension ("DDB Roll Sync")
- **What:** Captures your DDB Bearer token automatically and syncs new rolls (including **Pacts
  & Power**, `gameId 3661522`) to Supabase, auto-enriching each roll.
- **Triggerable actions (popup):**
  - **"⚡ Sync All Campaigns"** → fetch + upsert new rolls for every active campaign.
  - **"Clear"** → clear the popup log.
  - **Auto:** token capture runs in the background while you browse DDB.
- **Trigger:** click the extension icon in Chrome, then the button. (Load via
  `chrome://extensions` → Developer mode → Load unpacked → this folder.)
- **See also:** `ddb-roll-sync/README.md`.
