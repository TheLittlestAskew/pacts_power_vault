Section heading:
EcoSystem App Structure

Use this text:
When staff open the app, they see the shelter calendar. The calendar displays scheduled case meetings and can expand to include scheduled workers, staff meetings, and volunteer events.

Section heading:
Navigation

Use this text:
The navigation pane displays the areas each user has access to, including:

- Guests
- Case Meetings
- Budgets
- Resources
- Night Watch
- Reports

Section heading:
Views vs. Forms

Use this text:
Views display lists of records with basic information. Active views show current records; inactive views show historical records and discharged guests.

Forms are used for detailed information input and editing. Staff click “New” to create a record or select an existing record to open the form.

Section heading:
Case Meeting Procedures

Use this text:
Case meeting records document:

- Guest wins or accomplishments since the last meeting
- Challenges discussed
- Goals set
- Resources provided or coordinated
- Action items
- Phase advancement discussion, if applicable

After the meeting is documented, staff save the record and schedule the next meeting on the calendar.

Section heading:
Tracking Guest Wins

Use this text:
Guest wins are important for celebrating progress, grant reporting, building guest confidence, and demonstrating program effectiveness.

Document all wins, even small ones:

- Obtained ID
- Attended meeting
- Completed chore without reminder
- Made progress on goal
- Practiced new skill

Section heading:
Resource Assignment Procedures

Use this text:
When connecting a guest with resources, staff document:

- Resource type
- Agency or provider
- Contact information
- Referral date
- Follow-up requirements
- Outcome

Resource assignments are tracked in case notes.

Section heading:
Night Watch Documentation

Use this text:
Night Watch replaced the traditional paper Daily Log system. Evening, overnight, and weekend staff use Night Watch to document significant events during each shift.

Event types include:

- Daily Log
- Phone Calls
- Visitors
- Guest Interactions
- Meals
- Donations
- Medication Log
- Food / Item Requests
- Safety / Emergencies
- Intakes / Discharges
- Staff to Admin

Section heading:
Documentation Standards

Use this text:
Staff are instructed to document events as they happen, be specific and objective, note times when relevant, document both positive and concerning events, flag items needing follow-up, and review previous entries at the start of shift.
